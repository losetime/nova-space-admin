# 定时任务功能使用指南

## 功能概述

为 KeepTrack 元数据同步添加了定时任务功能，支持：
- ✅ 每小时自动触发同步
- ✅ 智能跳过（已有任务运行时）
- ✅ 前端界面开关控制
- ✅ 批量大小优化（60颗/批）
- ✅ 请求频率优化（10-40秒随机延迟）

## 默认状态

⚠️ **定时任务默认关闭**，需要手动在前端界面开启。

## 前端界面

### 位置

卫星同步页面：`/satellite-sync`

### 操作步骤

1. 登录管理后台
2. 进入"卫星同步"页面
3. 在"快速操作"卡片中找到"定时同步"开关

### 界面说明

```
┌─────────────────────────────────────────┐
│ 快速操作                                 │
├─────────────────────────────────────────┤
│ [完整同步（所有数据源）]  查看详情         │
│                                          │
│ ┌─────────────────────────────────────┐ │
│ │ 定时同步                     [开关] │ │
│ │ 每小时自动同步 KeepTrack 元数据      │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

- **开关状态**：
  - 开启（蓝色）：定时任务运行中，每小时整点触发同步
  - 关闭（灰色）：定时任务已禁用，不会自动触发（默认状态）

## 后端 API

### 1. 获取定时任务状态

```http
GET /api/satellite-sync/cron/status
```

**响应示例**：
```json
{
  "success": true,
  "data": {
    "enabled": false
  }
}
```

### 2. 切换定时任务开关

```http
POST /api/satellite-sync/cron/toggle
Content-Type: application/json

{
  "enabled": true
}
```

**响应示例**：
```json
{
  "success": true,
  "data": {
    "enabled": true,
    "message": "定时任务已启用"
  }
}
```

## 工作原理

### 定时任务执行流程

```
每小时整点触发
    ↓
检查定时任务开关
    ├─ 关闭 → 跳过执行
    └─ 开启 → 继续
        ↓
    检查当前是否有任务运行
        ├─ 有 → 跳过本轮，记录日志
        └─ 无 → 触发 keeptrack-meta 同步
            ↓
        每批处理 60 颗卫星
            ↓
        每次请求后等待 10-40 秒
```

### 批量处理策略

- **批量大小**：60 颗卫星/批
- **请求延迟**：10-40 秒随机延迟
- **预估速度**：约 25 分钟完成一批（60颗 × 平均25秒）

### 安全机制

1. **智能跳过**：已有任务运行时自动跳过，避免并发冲突
2. **频率控制**：随机延迟防止请求过于频繁
3. **API 配额保护**：保持在 KeepTrack API 限额内（2000次/小时）

## 使用场景

### 场景 1：启用自动同步

1. 进入卫星同步页面
2. 打开"定时同步"开关
3. 系统将每小时自动检查并同步

### 场景 2：API 配额不足时

如果 KeepTrack API 配额即将用完：

1. 关闭"定时同步"开关
2. 手动触发必要的同步任务
3. 等待配额恢复后再开启

### 场景 3：维护期间

系统维护或调试期间：

1. 关闭定时任务
2. 手动控制同步时机
3. 维护完成后重新开启

## 注意事项

1. **状态持久化**：定时任务开关状态保存在内存中，服务重启后会重置为默认关闭状态
2. **首次同步**：如果数据库中有大量卫星缺少元数据，需要多轮定时任务才能完成
3. **手动触发**：即使定时任务关闭，仍可手动触发同步
4. **并发控制**：定时任务会自动跳过已有任务运行的情况

## 日志说明

### 后端日志

定时任务执行时会在后端输出以下日志：

正常执行：
```
[SatelliteSyncService] [定时任务] 检查是否需要触发 KeepTrack 元数据同步...
[SatelliteSyncService] [定时任务] 开始触发 KeepTrack 元数据同步
```

跳过执行：
```
[SatelliteSyncService] [定时任务] 跳过本次同步，当前有任务正在运行: task-id (30/60)
```

定时任务禁用：
```
[SatelliteSyncService] [定时任务] KeepTrack 元数据同步已禁用，跳过
```
